## 3. 주요 기능 구현

### 3.1 현재 날씨 데이터 API 호출

* requests 라이브러리를 사용하여 OpenWeatherMap API 호출
* 응답 데이터를 JSON 형태(dict)로 변환

```python
response = requests.get(url)
data = response.json()
```

---

### 3.2 API 응답 Key 값 출력 (F101)

* API 응답 JSON의 **최상위 Key 값만 출력**하여 전체 구조 파악

```python
print(data.keys())
```

**출력 예시**

```
dict_keys(['coord', 'weather', 'base', 'main', 'visibility', 'wind', 'clouds', 'dt', 'sys', 'timezone', 'id', 'name', 'cod'])
```

---

### 3.3 특정 Key 값(main, weather) 추출 (F102)

* 전체 응답 중 `main`, `weather` Key만 추출하여 새로운 딕셔너리 구성

```python
filtered_data = {
    'main': data['main'],
    'weather': data['weather']
}
```

---

### 3.4 Key 값 한글 변환 (F103)

* 영문 Key를 한글로 변환하기 위한 매핑 딕셔너리 정의
* 새로운 한글 Key 기반 딕셔너리 생성

```python
key_map = {
    'coord': '좌표',
    'weather': '날씨',
    'main': '기온정보',
    'wind': '바람',
    'name': '도시이름',
    'cod': '응답코드'
}

korean_data = {}
for key, value in data.items():
    if key in key_map:
        korean_data[key_map[key]] = value
```

## 6. 단계별 구현 과정 및 회고

이번 과제는 단순히 API를 호출해서 값을 출력하는 것을 넘어서, **외부 데이터를 이해하고 가공하는 전체 흐름**을 직접 경험해볼 수 있었던 과정이었다. 단계별로 구현하면서 어떤 점을 배웠고, 어디에서 어려움을 느꼈는지, 그리고 어떤 생각이 들었는지를 중심으로 정리한다.

---

### 6.1 단계별 구현 과정에서 학습한 내용

#### (1) API 요청과 JSON 구조 파악

처음에는 API를 호출하면 바로 필요한 값이 나올 것이라고 막연히 생각했다. 하지만 실제로는 응답 데이터가 여러 단계로 중첩된 JSON 구조였고, 어떤 데이터가 어디에 들어 있는지를 먼저 파악하는 과정이 필요했다. 이 과정에서 `data.keys()`를 활용해 **전체 구조를 먼저 확인하는 습관**이 중요하다는 것을 배웠다.

#### (2) 필요한 데이터만 선별하는 과정

응답으로 내려오는 데이터는 생각보다 훨씬 많았고, 그중 실제로 필요한 정보는 일부에 불과했다. `main`, `weather`, 금융 API에서는 `baseList`, `optionList`처럼 **목적에 맞는 데이터만 골라 사용하는 것이 중요**하다는 점을 느꼈다. 이 과정을 통해 데이터를 무작정 쓰는 것이 아니라, 필요한 정보만 추려내는 관점이 생겼다.

---

### 6.2 구현 과정에서 어려웠던 부분

가장 어려웠던 점은 API 요청이 실패했을 때 원인을 파악하는 과정이었다. API Key가 잘못되었을 경우 정상 데이터가 아닌 `err_cd`, `err_msg`만 반환되는 상황을 겪었고, 이를 통해 단순한 코드 오류가 아니라 **요청 자체가 실패했을 가능성도 고려해야 한다는 점**을 배웠다.

또한 `optionList`가 리스트 안에 딕셔너리 형태로 구성되어 있어, 처음에는 어떤 방식으로 접근해야 할지 헷갈렸다. 반복문을 통해 하나씩 구조를 확인하면서 점차 익숙해질 수 있었다.

---

### 6.3 새롭게 배운 점

이번 과제를 통해 외부 API를 사용할 때는 반드시 공식 문서를 먼저 확인하고, 요청 변수와 응답 구조를 이해해야 한다는 것을 배웠다. 또한 동일한 데이터를 다양한 형태로 가공할 수 있으며, 데이터 구조를 어떻게 설계하느냐에 따라 활용도가 크게 달라진다는 점도 인상 깊었다.

---

### 6.4 과제를 진행하며 느낀 점

이번 과제를 통해 단순히 파이썬 문법을 아는 것과, **실제 데이터를 다루는 개발자 관점에서 사고하는 것 사이에는 큰 차이가 있다는 점**을 느꼈다. 처음에는 출력 결과를 맞추는 데에만 집중했지만, 점점 데이터의 구조와 의미를 함께 고민하게 되었다.
